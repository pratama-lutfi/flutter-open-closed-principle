# flutter-open-closed-principle
flutter code example represent open/closed principle

A part of SOLID principle series. 
## 0 = Open Close Principle

> "Open for extensions. Close for modification"


**Open for extensions**
	- We can still add new feature or behaviour in the existing component.
	- The existing component should be expandable

**Close for modification**
	- Don't touch stable function
	- Avoiding modification in the component function
